Categories
Mathematics>Sequences and bins

Signatures 
vec1d rgen(T i, j, n)

vec1d rgen_log(T i, j, n)

vec1d rgen_step(T i, j, s)

Description
The 
rgen()
functions generate a sequence of numbers that increase linearly. The first version generates integer values between 
i
and 
j
(inclusive) with step of 
1
. The second version generates 
n
floating point values between 
i
and 
j
(inclusive) with step 
(j-i)/(n-1.0)
.

The 
rgen_log()
function does the same as the second 
rgen()
function except that the steps are logarithmic (i.e., linear in logarithmic space) so that the size of the step increases continuously.

The 
rgen_step()
function does the same as the second 
rgen()
function except that the third argument is the step size, rather than the number of steps. This step size must be positive. The generated sequence is guaranteed to start with the value 
i
and end with the value 
j
. If 
abs(i-j)
cannot be evenly divided by the chosen step size, i.e., if 
i + n*s < j && j < i + (n+1)*s
(with 
n
an integer), then the last value of the sequence (
i + n*s
or 
i + (n+1)*s
, whichever is closest to 
j
) is replaced by the exact value of 
j
.

Example
rgen(1,4); // {1,2,3,4}
rgen(1,4,8); // {1,1.5,2,2.5,3,3.5,4}

rgen_log(1,4,8); // {1,1.219,1.486,1.811,2.208,2.692,3.281,4}

rgen_step(1,4,1); // {1,2,3,4}
rgen_step(1,3,0.3); // {1,1.3,1.6,1.9,2.2,2.5,2.8,3}
